## Architecture decision recorded

**`books_access` collapsed into `bookkeeper_scopes`.** The S60 plan
locked Option C (a separate `books_access` table for bookkeeper-to-
landlord linkage) before S80 shipped the team-permission consolidation
onto `bookkeeper_scopes`. Maintaining two parallel tables for the same
job — bookkeeper user × landlord client × access permission — would
have been pure cruft. S91 collapses: the 5 books bookkeeper-management
endpoints now read/write `bookkeeper_scopes`. `books_access` falls off
the phantom-table list without ever needing to exist.

**access_level (read_only | read_write) is the per-client granularity.**
The old `books_access.permissions jsonb` was a more flexible shape but
no consumer in the codebase used per-permission keys; the S80 model's
two-tier access_level covers what bookkeepers actually need. The
rewritten endpoints expose access_level under both `access_level` and
a synthetic `permissions: { access_level }` jsonb so any caller still
expecting the legacy shape gets a non-broken read.

**Direct-create `/invite` stays as a books-portal convenience.** The
canonical email-and-token invitation flow lives at
`POST /api/scopes/bookkeeper/invite` (S80). The books-portal
`/api/books/bookkeeper/invite` endpoint short-circuits that — it
takes a password directly and creates the user + scope rows in one
shot. Useful for admin-driven onboarding where you're sitting next to
the bookkeeper. For self-service invites (email link, accept page,
password set on accept), use the canonical scopes route.

**State-tax genericize is rename + strip, not a new feature.** The
column was already there with an AZ default; removing the AZ-isms
doesn't break any state's payroll math. New employees get 0% state
withholding by default (correct for TX/FL/NV/WA/etc — landlords
override per employee for states with income tax). Per-state tax form
catalog (CA DE-9, NY NYS-45, AZ A1-QRT, etc.) becomes a follow-up
product feature; for S91 the federal Form 941/940/W-2/1099-NEC
deadlines remain the only hardcoded list.

  UPDATE on (user_id, landlord_id) — idempotent re-assignment.
- `DELETE /bookkeeper/revoke` — hard delete on bookkeeper_scopes
  (S80 model has no soft-delete column).
